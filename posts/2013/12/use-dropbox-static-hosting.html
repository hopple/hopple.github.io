Using Dropbox For Hosting Static Files

Ganggao Zhu

on 2013 December 10 in Tutorial

As a developer, you may be familiar with the hosting service that costs you some money every year such as linode. The hosting service is ok, but it is inconvenient and expensive for small, simple sites, especially for personal usage. Suppose you have your own domain name from godadday and a dropbox account, you can use the public folder in your dropbox to host your static files such as images, pdfs and zips. Furthermore, you can configure your godadday domain service to forward the requests to the public address of your public folder in dropbox. In this way, you can have your own personalised domain to access your static files in dropbox rather than the odd dropbox default one. For example, suppose your id is 1111, then your static file has the public link address like below.

https://dl.dropboxusercontent.com/u/1111/some-static-file

In your godaddady domain service, you can create a subdomain like static.yourdomain.com, and configure the godaddady to forward all the requests to the dropbox public link https://dl.dropboxusercontent.com/u/1111/, then the link address of your static file will become the one as below.

static.yourdomain.com/some-static-file

In your godaddady domain manager, you can find the forwarding service like the one as below in the image.

Then you can create a subdomain and create the forwarding service like I did as below in the image. Choose the forwarding with masking so that your own domain will be shown all the time. Otherwise, the link will be changed into the dropbox one.

Finally, save your newly created configuration, and take a rest. Normally, it will take some time until it works.

Now, you have configured your own static file hosting service with your own personalised domain name. You can start to add folder and static files into your public folder of your dropbox. Then you can access them with the specific address.

You can also host your static blog in this way in dropbox, but I still recommend you to host in github. If you want to use dropbox for hosting, set those links as relative links if you use pelican to generate your static blog. Anyway, you can create a single web page and host it in dropbox if you want to make it have a specific subdomain.
