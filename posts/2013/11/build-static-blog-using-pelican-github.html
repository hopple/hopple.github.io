Building Static Blog Using Pelican and Github

Ganggao Zhu

on 2013 November 20 in Tutorial

Pelican is a python based static content generator like jekyll which can transform the plain text into static websites and blogs. Github provides static content hosting which can be used to host your static blog for free. I will talk about the idea behind the Pelcian first and then describe the main process of building a static blog with Pelican.

Basically, the static blog only contains static contents such as htmls, images, css files or javascript files. It does not have any database system to manage its contents, all the files are pure plain textual files. Differently, the WordPress blog system is a dynamic blog system since it needs a database system such as MySQL or PostgreSQL to store its contents.

The general idea of Pelican is to transform static Markdown or reStructuredText files into HTML files using Jinja template engine. I will explain the Pelican process by comparing to the process that you use Django to develop a blog system. While developing a blog system with Django, you need to create a database and design the table schemas first for storing your blog contents. Then you write your Django model for mapping your python object to the database records. In Pelican, it doesn't have database system, it requires you to write the schema information ( title, date, category, tag, author etc.) in the beginning of the file. You need to follow the rules provided by Pelican in writing and storing files so that Pelican can read the data in your file into their inner python models.

Having the models, you will need to design the views by using the Django templates. Pelican use Jinjia to do the same work since Jinjia is a similar template engine as the one in Django. Therefore, if you want to change your views, you need to modify the template files as well. And if you are familiar with the Django template engine, you should be fine with the Jinjia template engine as well. Anyway, Pelican has implemented the basic templates and themes for you to start your sites or blogs easily. However, if you want to personalize your site, you need to have some basic knowledge of Jinja template engine and the Pelican settings.

Now, let us start the journey with Pelican. Actually, it is extremely easy. You can refer to Pelican Get Started Doc for more details. Here I will describe my own experience in using Pelican. There is also a quick video cast for you to know the whole process.

I suppose you have installed python and pip. There are some folks suggesting to use VirtualEnv, but it depends on your choice. And also I will use Markdown to write my articles, so I will install Markdown module as well. Finally, just tpye the following commands to install the pelican and Markdown if you also want to use it to write your article. 

pip install pelican Markdown

Then you should go to your wokring directory and start buidling your own site by running Pelican's quickstart.

pelican-quickstart
yourproject/
├── content
│   └── (pages)
├── output
├── develop_server.sh
├── fabfile.py
├── Makefile
├── pelicanconf.py       # the main setting file
└── publishconf.py       # for publishing

Follow the guide provide by the quickstart and if everything is ok, you should get a directory containing files like above. You need to store your blog articles inside the content folder. Specifically, the "pages" folder is the place where you store your page files like "about page". Put your contents in proper folder so that Pelican can transform them correctly. Try to have your own first post article and store it in side the content folder, then you can run the Pelican command as below.

pelican content -o <output location> -s pelicanconf.py
cd output
python -m SimpleHTTPServer

If you don't specify the output location, your web site will be saved in the output folder. In my case, I use the default output for viewing locally. If I need to publish my blog, I will just specify the output to my Github hosting folder. Pelican provides the basic theme, so you can start viewing your site by using python's built in web server. If you want to have your own theme, you need to follow How to create themes for pelican guide.

When you want to publish your blog online, you can start with Github's static content hosting service. Just create a Github repository with your github account and name it with yourusername.github.io, so the github knows this is your personal page. You can just push your whole output folder to that repository. Then your can view your blog with the url address of yourusername.github.io. You can also have your own domain name. For the details of hosting your blog in Github, refer to Github Page Help.

I need to thank the Pelican and Github, because I could start this blog so easily in such a low cost. Why don't you start building your own blog right now?
